Banner Component

The Banner component renders a responsive banner which shows different views for desktop and mobile, exactly as implemented in the Curative UI design system using Headless UI Disclosure.

Responsive Banner (Auto)

This banner automatically adapts to screen size - desktop shows static content, mobile shows collapsible disclosure.

Note: Resize your browser window to see the responsive behavior. Desktop view shows at 768px+ width, mobile view shows below 768px.

Desktop View (Forced)

This shows how the banner appears on desktop screens (768px+) with static content display.

Desktop view always shows the content directly without any toggle functionality. This provides immediate visibility of important information on larger screens.

Mobile View (Forced)

This shows how the banner appears on mobile screens (<768px) with collapsible disclosure functionality.

This content is initially hidden on mobile and can be revealed by tapping the banner toggle. This helps save space on smaller screens while keeping important information accessible.

This content is initially visible on mobile (shouldStartOpen=true). Users can collapse it by tapping the banner toggle to save screen space when desired.

Usage

How to implement the Banner component with the exact Curative UI structure.

Props Interface

export type BannerProps = CUIComponentProps< PropsWithChildren<{ title: string; shouldStartOpen?: boolean; className?: string; textVariant?: TextVariant; }> >;

Desktop HTML Structure

<div class="banner"> <div class="banner--desktop"> <div class="banner__content"> {children} </div> </div> </div>

Mobile HTML Structure

<div class="banner"> <div class="banner--mobile"> <button class="banner__toggle" aria-expanded="true|false"> <div class="banner__toggle-left"> <span class="banner__toggle-icon"> <span class="heroicon heroicon-16 heroicon-exclamation-circle"></span> </span> <span class="banner__toggle-title">{title}</span> </div> <span class="banner__toggle-chevron"> <span class="heroicon heroicon-16 heroicon-chevron-up|down"></span> </span> </button> <div class="banner__panel banner__panel--visible|hidden"> <div class="banner__panel-content"> {children} </div> </div> </div> </div>

Responsive Behavior

/* Desktop: 768px+ (desktopBreakpoint = BREAKPOINT_MD) */ const desktop = useMediaPredicate(`(min-width: 768px)`); // Desktop View: - Shows content directly without toggle - Uses: justify-center p-4 pt-3 // Mobile View: - Shows collapsible disclosure with toggle - Uses: flex-col py-2 px-4 - shouldStartOpen controls initial state

Tailwind Class Mappings

/* Base Banner Classes */ .banner { display: flex; /* flex */ border-top: 4px solid; /* border-t-4 */ border-color: var(--color-border-control); /* border-t-borderColor-control */ background-color: var(--color-surface-highlight); /* bg-surface-highlight */ } /* Desktop */ .banner--desktop { justify-content: center; /* justify-center */ padding: 16px; /* p-4 */ padding-top: 12px; /* pt-3 */ } /* Mobile */ .banner--mobile { flex-direction: column; /* flex-col */ padding: 8px 16px; /* py-2 px-4 */ } /* Toggle */ .banner__toggle { display: flex; /* flex */ width: 100%; /* w-full */ justify-content: space-between; /* justify-between */ } /* Text Color */ .banner__content, .banner__toggle-title, .banner__panel-content { color: var(--color-text-highlight-default); /* text-textColor-highlight-default */ } /* Icon Sizes */ .banner__toggle-icon, .banner__toggle-chevron { width: 16px; /* SpacingVariant.S16 */ height: 16px; } /* Spacing */ .banner__toggle-title { margin-left: 8px; /* margin: { left: SpacingVariant.S8 } */ } .banner__panel { margin-top: 8px; /* mt-2 */ } .banner__panel-content { margin-left: 24px; /* margin: { left: SpacingVariant.S24 } */ padding-right: 16px; /* padding: { right: SpacingVariant.S16 } */ }
