How to Enable Side Navigation

Add the carousel--side-nav class to the carousel container:

<div class="carousel carousel--side-nav">
    <div class="carousel__track">
        <!-- slides -->
    </div>
    <div class="carousel__controls">
        <div class="carousel__nav">
            <button class="carousel__button" data-carousel-prev>
                <span class="carousel__button-text">Previous</span>
                <span class="heroicon heroicon-20 heroicon-chevron-left"></span>
            </button>
            <button class="carousel__button" data-carousel-next>
                <span class="heroicon heroicon-20 heroicon-chevron-right"></span>
                <span class="carousel__button-text">Next</span>
            </button>
        </div>
    </div>
</div>

With Bottom Indicators (Optional)

Add carousel--show-indicators class and create separate indicator elements below:

<div class="carousel carousel--side-nav carousel--show-indicators">
    <!-- carousel content -->
    <div class="carousel__indicators-bottom">
        <button class="carousel__indicator"></button>
        <!-- more indicators -->
    </div>
    <div class="carousel__counter-bottom">1 / 4</div>
</div>

In-Slide Navigation Attributes

Add these data attributes to any element inside a slide to enable navigation:

data-carousel-next-slide

Navigate to the next slide when clicked:

<button class="button button--primary" data-carousel-next-slide>
    Continue
</button>

data-carousel-prev-slide

Navigate to the previous slide when clicked:

<button class="button button--outline" data-carousel-prev-slide>
    Go Back
</button>

data-carousel-goto-slide="index"

Navigate to a specific slide by index (0-based):

<button class="button button--outline" data-carousel-goto-slide="0">
    Start Over
</button>

<button class="button button--primary" data-carousel-goto-slide="3">
    Skip to Results
</button>

Works on Any Element

These attributes work on any clickable element inside a slide:

<!-- Buttons -->
<button data-carousel-next-slide>Next</button>

<!-- Links -->
<a href="#" data-carousel-next-slide>Learn More</a>

<!-- Cards/Divs -->
<div class="card" data-carousel-next-slide style="cursor: pointer;">
    <h3>Click to continue</h3>
</div>

<!-- Images -->
<img src="image.jpg" data-carousel-next-slide style="cursor: pointer;" alt="Click to advance">

Use Cases

  • Onboarding flows with "Get Started" or "Next" buttons in slides
  • Interactive tutorials with step-by-step navigation
  • Product showcases with "Learn More" call-to-actions
  • Clickable card carousels where entire cards advance slides
  • Image galleries with clickable images to advance

How to Enable Track Views

Add two data attributes to your carousel:

  • data-carousel-track-views - Enables slide view tracking
  • data-carousel-controlled-button="buttonId" - ID of the button or link to control

For Button Elements

The controlled button should have disabled attribute initially:

<div class="carousel" 
     data-carousel-track-views 
     data-carousel-controlled-button="myButton">
    <!-- carousel content -->
</div>

<button id="myButton" class="button button--primary" disabled>
    Continue
</button>

For Anchor Elements

For anchor tags, the script automatically adds the appropriate disabled class:

<div class="carousel" 
     data-carousel-track-views 
     data-carousel-controlled-button="continueLink">
    <!-- carousel content -->
</div>

<a id="continueLink" 
   href="/next-page/" 
   class="button button--big-primary">
    Continue
</a>

Note: The script automatically detects the button variant (button--primary, button--big-primary, or button--small-primary) and applies the correct disabled class (-disabled suffix) for anchor tags.

Accessibility Features

The Carousel component includes comprehensive accessibility support.

Navigation Behavior

  • Previous Button: Hidden when on first slide
  • Next Button: Hidden when on last slide
  • Indicators: Show current slide position with active state
  • Counter: Displays "current / total" slide numbers
  • Auto Scroll: Automatically scrolls to the top of the carousel when navigating to a new slide (with 124px offset for headers)
  • Staggered Animations: Exit animation (fade out + slide left, 300ms) completes before entry animation (fade in + scale up, 400ms) begins

Keyboard Navigation

  • Left Arrow: Navigate to previous slide
  • Right Arrow: Navigate to next slide
  • Tab: Move focus between interactive elements
  • Enter/Space: Activate focused button or indicator

ARIA Attributes

  • role="region" on carousel container
  • aria-label describing carousel purpose
  • aria-hidden on non-active slides
  • aria-live="polite" on slide counter for announcements
  • aria-current on active indicator

JavaScript API

Control carousel behavior programmatically using the JavaScript API.

API Methods

Initialize Carousel

initializeCarousel();

Navigate to Next Slide

carouselNext('carouselId');

Navigate to Previous Slide

carouselPrev('carouselId');

Go to Specific Slide

carouselGoTo('carouselId', slideIndex);

Access Carousel API

const carousel = document.getElementById('myCarousel');
carousel.carouselAPI.next();     // Go to next slide
carousel.carouselAPI.prev();     // Go to previous slide
carousel.carouselAPI.goTo(2);    // Go to slide index 2
carousel.carouselAPI.getCurrentIndex(); // Get current slide index
carousel.carouselAPI.getTotalSlides();  // Get total slide count
carousel.carouselAPI.getViewedSlides(); // Get array of viewed slide indices
carousel.carouselAPI.areAllSlidesViewed(); // Check if all slides viewed
