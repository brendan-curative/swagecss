Foundation Integration

All BLV components are built using the Swage CSS foundation system.

Foundation Compliance

✅ Colors

  • • All colors use foundation variables
  • • Semantic color naming (primary, success, etc.)
  • • No hardcoded hex values
  • • Consistent theming support

✅ Typography

  • • Foundation font family (Figtree)
  • • Consistent font sizes and line heights
  • • Semantic typography classes
  • • Responsive text scaling

✅ Spacing

  • • 4px base grid system
  • • Foundation spacing variables
  • • Consistent margins and padding
  • • Logical spacing relationships

✅ Layout

  • • Flexbox and grid utilities
  • • Responsive design patterns
  • • Foundation display classes
  • • Mobile-first approach

Usage Guidelines

Best practices for implementing and using BLV components.

Important Note

These components are specifically designed for baseline visit workflows and replicate the exact behavior and styling of the original implementation while using foundation CSS variables.

Implementation Guidelines

CSS Import

All BLV components are automatically included when importing the main CSS:

@import 'src/swage.css';

JavaScript Initialization

Components are automatically initialized on page load. Individual component scripts are included:

src/components/blv/horizontal-stepper.js
src/components/blv/checkmark-button.js
src/components/blv/rating-picker.js
src/components/blv/checkmark-label.js
src/components/blv/ordered-list.js
src/components/blv/radio-selection-list.js
src/components/blv/stat-card.css

Accessibility

All components include:

  • • Proper ARIA attributes and roles
  • • Keyboard navigation support
  • • Focus management
  • • Screen reader compatibility
  • • High contrast mode support
