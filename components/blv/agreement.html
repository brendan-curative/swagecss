Overview

The Agreement component displays large bodies of text in a scrollable container with a checkbox for user agreement. Designed for terms of service, privacy policies, and other legal agreements in the BLV prototype theme.

Key Features

  • Scrollable content area with fixed height (300px)
  • Control Short Variant checkbox styling
  • Footer button disabled until all agreements are checked
  • Supports multiple agreements on one page
  • BLV theme with glassmorphism effects

Front Matter Setup

Enable the component by adding agreement-enabled: true to your page's front matter:

---
layout: templates/blv/layout.html
title: Terms and Conditions
agreement-enabled: true
---

CSS Classes

Main Classes

  • .agreement - Container for the entire agreement section
  • .agreement__content - Scrollable content area
  • .agreement__button - Checkmark button wrapper (inherits from .checkmark-button)
  • .checkmark-button - Base checkmark button styling (BLV component)
  • .checkmark-button__icon - Checkmark icon SVG
  • .checkmark-button__text - Button text label

State Classes

  • .checkmark-button--checked - Applied when button is checked
  • data-checked="true" - Data attribute indicating checked state
  • data-checked="false" - Data attribute indicating unchecked state

Behavior

Footer Button State Management

  • Initial State: Button is disabled (button--big-primary-disabled)
  • Tracking: Component monitors all agreement checkmark buttons on the page
  • Enabled: Button becomes clickable once ALL agreements are checked
  • Re-disabled: Button is disabled again if any button is unchecked

Checkmark Button Interaction

  1. Unchecked: Gray border, gray text, empty circle icon
  2. Hover: Visual feedback on hover state
  3. Checked: Primary color border and text, filled circle with checkmark
  4. Click: Toggles between checked and unchecked states
