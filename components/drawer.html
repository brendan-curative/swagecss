The Drawer component provides a slide-out panel that appears over the existing page content. It supports left and right positioning, multiple sizes, keyboard navigation, and full accessibility features.

✨ Plugin Architecture

The Drawer is implemented as a standalone plugin with automatic initialization, focus management, and accessibility features built-in.

Implementation Guide

How to implement and customize drawer components.

Basic HTML Structure

<!-- Trigger Button -->
<button data-drawer-target="my-drawer">Open Drawer</button>

<!-- Drawer Container -->
<div id="my-drawer" data-drawer data-drawer-position="left" data-drawer-size="medium">
  <div class="drawer__header">
    <h3 class="drawer__title">Drawer Title</h3>
    <button class="drawer__close" aria-label="Close drawer">
      <span class="heroicon heroicon-x-mark"></span>
    </button>
  </div>
  
  <div class="drawer__content">
    <!-- Your content here -->
  </div>
  
  <div class="drawer__footer">
    <!-- Optional footer content -->
  </div>
</div>

Configuration Options

Drawer Container Attributes

  • data-drawer - Required: Marks element as a drawer
  • data-drawer-position - "left" or "right" (default: "left")
  • data-drawer-size - "small", "medium", "large", "full" (default: "medium")
  • data-drawer-close-on-overlay - "true" or "false" (default: "true")
  • data-drawer-close-on-escape - "true" or "false" (default: "true")
  • data-drawer-focus-trap - "true" or "false" (default: "true")
  • data-drawer-auto-focus - "true" or "false" (default: "true")

Trigger Button Attributes

  • data-drawer-target - Required: ID of the drawer to control

JavaScript API

// Open drawer
openDrawer('my-drawer');

// Close drawer
closeDrawer('my-drawer');

// Toggle drawer
toggleDrawer('my-drawer');

// Position override example
// Trigger buttons automatically handle position changes
// No manual JavaScript needed for trigger-based positioning

// Event listeners
document.addEventListener('drawer:open', (e) => {
  console.log('Drawer opened:', e.detail.drawer);
  console.log('Current position:', e.detail.drawer.options.position);
});

document.addEventListener('drawer:close', (e) => {
  console.log('Drawer closed:', e.detail.drawer);
});

Accessibility Features

Built-in accessibility support for screen readers and keyboard navigation.

Keyboard Navigation
  • • Escape - Close drawer
  • • Tab - Navigate forward through focusable elements
  • • Shift+Tab - Navigate backward through focusable elements
  • • Focus trap keeps focus within drawer when open
Screen Reader Support
  • • role="dialog" for proper announcement
  • • aria-modal="true" when open
  • • aria-hidden state management
  • • aria-expanded on trigger buttons
  • • Automatic focus management
Focus Management
  • • Auto-focus first interactive element when opened
  • • Return focus to trigger when closed
  • • Focus trap within drawer content
  • • Skip to close button with proper labeling
